Detailed Description

template<unsigned int dim>
class coupling::solvers::FoamSolverInterface< dim >

interface to the Foam solver. It's almost the same as the CouetteSolverInterface. That means, data from MD is received in the inner region plus one cell layer from the outer region. Plus one is necessary cause the Foam solver needs the boundary condition on the boundary. So it is interpolated between the two cells beside. The Foam solver sends data in tha outerRegion region, as defined by the parameters of the interface. . By default, the argument outerRegion is set to 1: this yields that only the first non-ghost layer of macroscopic cells shall be sent to MD, and all other inner macroscopic cells are received from MD. The Foam solver is expected to only run on rank 0.

virtual std::vector< unsigned int > coupling::solvers::FoamSolverInterface< dim >::getRanks ( tarch::la::Vector< dim, unsigned int >  globalCellIndex)
inlinevirtual

This function determines all the ranks on which the macroscopic solver holds data of the macroscopic cell at index globalCellIndex. By default, this method is used for send/recv operations. However, there are situations where the target and source rank definitions (target rank=rank of macroscopic solver that shall receive data from MD; source rank=rank of macroscopic solver that shall send data to MD) may differ. For example, assume a domain decomposition using ghost layers. You may want to send data from cells inside a process-local domain and want to receive data in all copies of a given macroscopic cell (incl. ghost cells), but you may not be able to provide valid data on every process for each cell instance (e.g., if this cell is part of a ghost layer). For this case, the method getSourceRanks() and getTargetRanks() can be implemented accordingly.
